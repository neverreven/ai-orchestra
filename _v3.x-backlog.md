# AI Orchestra v3.x — Backlog

> Ideas, deferred features, and planned improvements for the v3 ensemble.
> Items are roughly grouped by theme. No priority ordering — this is a capture list, not a sprint plan.

---

## Runtime reliability

- [ ] **Health monitor** — Lead agent pings role agents on a heartbeat interval; auto-restarts unresponsive bots via their registered PID
- [ ] **Task timeout** — configurable per-task deadline; Lead escalates if a role agent doesn't report within N minutes
- [ ] **Retry logic** — automatic retry with exponential backoff for transient Telegram API errors
- [ ] **Docker / docker-compose** — single-command local deployment of the full agent team
- [ ] **Dockerfile** — production-ready image for VPS / Railway / Fly.io deployment

## Telegram UX

- [ ] **Inline status buttons** — `/status` shows per-agent health as inline buttons that link to recent log excerpts
- [ ] **Task feed channel** — optional Telegram channel that receives task completion summaries (one post per completed task)
- [ ] **Voice message support** — transcribe voice notes to text before routing to Lead
- [ ] **File upload handling** — accept documents/images from user, save to project CWD, include in task context

## Agent capabilities

- [ ] **Model routing at task level** — Lead decides which model tier to use per delegated task based on complexity estimate
- [ ] **Context window management** — truncate/summarise conversation history when approaching model context limit
- [ ] **Multi-project support** — a single Lead agent managing multiple CWDs (switched via `/project <name>`)
- [ ] **Continuous learning loop** — Lead writes to `AI_LEARNINGS.md` and `CHANGELOG.md` after each completed task

## Security & scope

- [ ] **Scope violation alerts** — notify owner via Telegram when any agent triggers a `forbidden` scope violation
- [ ] **Audit log export** — `/export-log` command produces a downloadable JSON of the access log
- [ ] **Dependency scan integration** — Security agent runs `npm audit` / `pip-audit` / `cargo audit` on demand

## OpenSpec

- [ ] **Status-update schema** — formal JSON Schema for the `status-update` bus message type
- [ ] **Validation middleware** — validate all bus messages against schemas at write time (using Zod or Ajv)
- [ ] **Schema versioning** — add `schemaVersion` to `bus-message.schema.json` for forward compatibility

## Developer experience

- [ ] **`bun run new <role>`** — scaffold a new custom role agent from a prompt
- [ ] **`bun run typecheck`** — full workspace type-check script
- [ ] **Unit tests** — test suite for `scope.ts`, `bus.ts`, `auth.ts` edge cases
- [ ] **`.cursor/rules/` injection** — Lead can propose updates to Cursor rules based on learnings

## Spec layer (v3.x)

- [ ] **`{{INSTALLED_FOLDER}}` variable in adapter render-rules** — all adapter `render-rules.md`, `target-schema.md`, and `mappings.md` files currently hardcode `score/` (or `ai-orchestra/` in older entries) for skill catalog entries, stop-hook strings, and post-install checks. Replace with a `{{INSTALLED_FOLDER}}` placeholder that the adapter substitutes from `install.json`.`installedFolder` at render time. This enables the same adapter templates to work correctly for both v3 (`score/`) and legacy (`ai-orchestra/`) installs without needing a separate upgrade path for rendered output.
- [x] **Codex/VS Code skill catalog entries** — updated to `{{INSTALLED_FOLDER}}/core/skills/...` across all adapter files (v3.0.1).
- [x] **`_lint.md` URL hygiene rule** — updated to reference `score/core/` with legacy note (v3.0.1).

## Integrations

- [ ] **GitHub integration** — role agents open PRs and request reviews via GitHub API after completing tasks
- [ ] **Slack bot variant** — grammy → Slack Bolt adapter for teams that use Slack instead of Telegram
- [ ] **Linear / Jira** — task creation and status sync for teams with existing issue trackers
